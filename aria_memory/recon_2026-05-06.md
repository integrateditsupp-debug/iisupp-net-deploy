## 2. Function Inventory (19)

- `aria` → `/api/aria` — main brain. Anthropic v2 API, model `claude-sonnet-4-6`, web_search tool enabled, JSON output contract `{intent, response, format, escalate, phone, next_step}`. Used by `aria.html` chat.

- `aria-license` — issues / verifies / replaces license tokens. 5-device cap. Stores in Blobs `aria-licenses`.

- `aria-search` — DuckDuckGo Instant Answer wrapper, no auth needed.

- `stripe-confirm` — retrieves session, calls `aria-license` to issue token, fires Resend welcome email with token + activation steps.
- `stripe-webhook` — Stripe webhook receiver.

## 7. Critical Issues (ranked)

### P0 — REVENUE LEAK (index.html PayPal sandbox)
`index.html:16` loads PayPal SDK with `client-id=sb` — that's PayPal's **public sandbox client ID**. Every "purchase" on the technical-services PayPal flow on the homepage is **not collecting real money.** This is for the index.html purchase modal (line 2461+ "About to purchase X for $X CAD"). If anyone has clicked through, they got a sandbox checkout, not a real charge. **Fix is one env var swap + one script src change. Zero visual impact.**

### P1 — ARIA brain fragmentation
Two endpoints (`/api/aria` via aria.mts, and unrouted aria-chat) with **incompatible JSON contracts and different models**. Web uses aria.mts. aria-chat has the emotional-intelligence prompt and learning-loop wiring — but isn't used by the website. Either: (a) consolidate into one (recommend), or (b) decide which is the "voice/phone brain" vs "web/text brain" and document. **Picking one is a Garry Tan move — clarity in 5 seconds.**

### P1 — Voice agent missing webhooks
`aria-sms`, `aria-call-summary`, `aria-caller-lookup` referenced in tools-spec.json but not deployed. If you turn on ElevenLabs voice agent today, the tools fail silently.

### P1 — Web chat doesn't feed learning loop
aria.mts (the live web brain) doesn't call aria-learn. Self-learning data only comes from aria-chat.js, which is unrouted from aria.html. So **the learning loop is empty** until either (a) aria.mts is updated to log, or (b) aria.html switches to aria-chat.

### P2 — Uptime monitoring not live
Site is up now, but no continuous check. Need a Netlify scheduled function (free) hitting `/.netlify/functions/health` every N minutes, with Resend alert on 3 consecutive failures.

### P2 — `www.` vs apex
Voice-agent webhooks reference `https://www.iisupp.net/...`. Primary URL is apex `https://iisupp.net`. Need to confirm www → apex redirect is configured (Netlify usually auto-handles, but worth checking).

### P3 — Anthropic API key not verifiable from sandbox
Code references `ANTHROPIC_API_KEY`. Can't hit live `/.netlify/functions/health` from this environment to confirm it's set. Either: (a) Ahmad opens iisupp.net/.netlify/functions/health in his browser and confirms `"anthropic": true`, or (b) once Phase 1 monitor is live, it'll surface this on first run.
